# Введение в разработку с Antigravity

Добро пожаловать в проект создания вашего сайта-визитки! Этот документ описывает базовый процесс работы, инфраструктуру и правила взаимодействия с ИИ-агентом (Antigravity).

## 1. Архитектура и Инфраструктура

Поскольку это сайт-визитка, мы рекомендуем следующий стек для начала, так как он обеспечивает максимальную производительность, красивые анимации и отличный SEO:
*   **Основа:** HTML5 + Vanilla JS (или легковесный фреймворк вроде Vite + React/Vue, если планируется сложная логика).
*   **Стилизация:** Vanilla CSS (современный CSS с переменными и flexbox/grid) или TailwindCSS (если вам так удобнее).
*   **Деплой:** Ваш собственный сервер (VPS) по адресу `64.188.56.128`. Настроен SSH-доступ без пароля по ключам (`id_ed25519_server`), добавлен удобный alias `myserver` в `~/.ssh/config`. Для раздачи статики планируется использовать Nginx.
*   **Контроль версий:** Локальный Git-репозиторий инициализирован и привязан к удаленному GitHub [msinskas/Test-project-for-antigravity](https://github.com/msinskas/Test-project-for-antigravity). Настроена интеграция MCP (с использованием Personal Access Token) для автономной работы агента с историей коммитов.

## 2. Жизненный цикл разработки (Workflow)

Процесс разделен на три основных этапа:

### Этап 1: Локальная разработка (Local)
1.  **Постановка задачи:** Вы передаете агенту требования (например, макет из Figma, текст, описание логики).
2.  **Генерация кода:** Агент пишет код, создает структуру проекта (HTML, CSS, JS, ассеты).
3.  **Локальное тестирование:** Агент (или вы) запускает локальный сервер (например, через `npm run dev` или простой HTTP-сервер). Мы проверяем, как сайт выглядит в браузере. Вы можете использовать инструмент `browser_subagent` для того, чтобы агент сам посмотрел на результат и сделал скриншоты.
4.  **Итерации:** Вы даете правки ("сделай кнопку крупнее", "измени цвет фона"), агент вносит изменения.

### Этап 2: Тестирование и Сборка (Test & Build)
1.  **Проверка качества:** Агент проверяет код на ошибки (линтеры), адаптивность (мобильные устройства) и SEO-теги.
2.  **Фиксация версий (Git):** После завершения логического блока функционала, агент делает коммит в Git (через MCP).
3.  **Оптимизация:** Минификация CSS/JS, сжатие изображений для быстрой загрузки.

### Этап 3: Деплой на Сервер (Deploy)
1.  Когда локально всё устраивает, код переносится на сервер.
2.  Агент может подключиться к серверу по SSH (ключ мы сгенерировали ранее), клонировать туда репозиторий (или скопировать файлы по scp/rsync) и перезапустить веб-сервер (Nginx).
3.  (В будущем можно настроить CI/CD через GitHub Actions или GitLab CI для автоматического деплоя при пуше в `main` ветку).

## 3. Правила работы с агентом (Antigravity)

Чтобы наше сотрудничество было максимально эффективным, соблюдайте следующие правила:

1.  **Давайте четкие задачи:** Чем конкретнее описание (цвета, шрифты, отступы, логика), тем точнее результат. Разделяйте большие задачи на мелкие шаги.
2.  **Используйте "Task Boundary":** Агент обучен работать в рамках задач. Если задача сложная, агент создаст План (Planning), попросит вас его утвердить, а затем перейдет к Выполнению (Execution).
3.  **Не стесняйтесь просить переделать:** Если дизайн выглядит "дешево", прямо скажите: "Сделай дизайн более премиальным, добавь градиенты и микро-анимации". Агент знает современные тренды.
4.  **Спрашивайте причину:** Если агент предлагает незнакомую технологию, спросите "почему это лучше?".
5.  **Ревью изменений:** Перед сохранением больших изменений или выполнением команд в терминале, агент всегда спрашивает ваше разрешение. Читайте, что именно будет выполнено.

### Браузер для тестирования
*   **Локальный просмотр:** Вы можете использовать ваш привычный браузер (к примеру, Brave) для просмотра в реальном времени.
*   **Тестирование Агентом:** Агент (Antigravity) использует свою внутреннюю, изолированную среду на базе движка Chromium. Это обеспечивает безопасность ваших личных данных и сохраняет чистую среду для тестов.
