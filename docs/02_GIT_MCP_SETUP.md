# Настройка интеграции с Git через MCP (Model Context Protocol)

Для того чтобы я (агент Antigravity) мог полноценно работать с вашим Git-репозиторием локально (читать историю, создавать коммиты, переключать ветки), нам нужно настроить MCP-сервер для Git.

## Что такое MCP для Git?

MCP (Model Context Protocol) позволяет ИИ безопасно вызывать локальные инструменты. Официальный сервер Git MCP предоставляет агенту набор действий: `git_status`, `git_diff`, `git_commit`, `git_log` и т.д.

## Шаги по настройке (Используем Claude/Cursor/Antigravity стандарты)

Обычно настройка MCP требует добавления конфигурации в настройки вашего ИИ-редактора (если вы используете среду, поддерживающую MCP, например Cursor или подобные интерфейсы).

### Вариант 1: Использование `npx` (Рекомендуемый)

Убедитесь, что у вас установлен **Node.js** (и `npx` доступен в терминале). Если нет, скачайте его сnodejs.org.

В конфигурационный файл вашего MCP-клиента добавьте следующую запись:

```json
{
  "mcpServers": {
    "git": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-git"],
      "env": {}
    }
  }
}
```

*   Этот сервер автоматически определит `.git` директорию в корне нашего проекта (`/Users/maximsinskas/Projects/Test project/`) и позволит мне выполнять команды.

### Вариант 2: Использование Python (Альтернатива)

Если вы предпочитаете Python, существует `mcp-server-git` пакет:
```bash
# Установка через pip/uv
pip install mcp-server-git
# ИЛИ
uvx mcp-server-git
```

В конфигурации MCP:
```json
{
  "mcpServers": {
    "git": {
      "command": "uvx",
      "args": ["mcp-server-git"],
      "env": {}
    }
  }
}
```

## Как это изменит наш процесс работы?

1.  **Создание веток:** Перед началом работы над новой фичей (например, "добавление секции Портфолио"), я смогу создать ветку `feature/portfolio`.
2.  **Коммиты:** После завершения задачи я сделаю атомарный коммит: `git commit -m "feat: adding portfolio section with animations"`.
3.  **Автоматизация:** Мне больше не нужно просить вас вводить `git add .` и `git commit` — я всё сделаю сам через безопасный протокол, предварительно показав вам `diff`.

## Что сделать прямо сейчас?

1.  Убедитесь, что папка проекта инициализирована как Git-репозиторий: `git init`.
2.  Добавьте конфигурацию MCP-сервера в настройки вашего ИИ-агента.
3.  После этого я автоматически получу доступ к инструментам `git_log`, `git_commit` и др.
